## 1. デモシェル — 全デモ共通の 6 機能

すべてのデモは共通シェル(1 実装)+ シーン JSON + デモ固有パネル(宣言的定義)で構成する。
共通 6 機能:

| # | 機能 | 使う World API |
|---|---|---|
| 1 | **再生制御**: 再生 / 一時停止 / 1 step / スロー・加速(倍率常時表示、[20-integration/06](../20-integration/06-regime-switching.md)) | `step()` 呼び出し制御、`SetTimeScale` コマンド |
| 2 | **Probe グラフ + CSV エクスポート**: 任意観測量の時系列グラフ(複数系列・対数軸・フィット線) | `Probe`([20-integration/04](../20-integration/04-world-api.md) §2.1) |
| 3 | **保存量パネル**: エネルギー台帳の内訳・残差トレンド・運動量/角運動量 | `energy_ledger()` `total_momentum()` |
| 4 | **シード・ハッシュ・リプレイ**: シード表示/変更、状態ハッシュ常時表示、リプレイの保存/読込/再生 | `state_hash()` `snapshot()` `start_replay_recording()` |
| 5 | **近似バッジ**: このシーンで効いている近似の一覧(クリックで設計文書の該当節へリンク) | `approximations()` |
| 6 | **診断表示**: 発散警告・CFL 違反・シーンクラス(S/M/L)とスロー再生バッジ([00-foundation/05](../00-foundation/05-rust-wasm-platform.md) §5.1) | `diagnostics()` |

- シェルは**ヘッドレスでも動く**: UI なしで シーン JSON + 入力列 + Probe assert を実行できる
  ランナーを同梱し、デモ合格基準の CI テスト(D1–D43)はこのランナーで回す(§5)。

## 2. 予測 → 実験ワークフロー(画面遷移)

「検証して遊ぶ」の中核導線。全デモで同一の 4 画面遷移:

```text
[1] シーン確認    シーンとパラメータ(スライダー)を見る・変える
      ↓「予測する」
[2] 予測入力     問い(例: 落下時間は?)に対し、数値入力 or 選択肢 or 式
                  (解析式パネルに理論式と代入値を並記 — 教材モードでは隠せる)
      ↓「実験開始」
[3] 実行・観測    シェルの再生制御 + Probe グラフに「予測線」を重ね描き
      ↓ 自動停止(Probe の停止条件)or 手動停止
[4] 比較・振り返り 実測値 vs 予測値 vs 解析解(誤差%)。「もう一度」(同シード)/
                  「条件を変えて」(パラメータへ戻る)/「シードを変えて」
```

- 予測はスキップ可能(自由に遊ぶモード)。予測データはローカル保存のみ(共有はリプレイ+
  シーン JSON で行う — [20-integration/02](../20-integration/02-determinism-replay.md) §4)。
- 実装は宣言的: 各デモの「問い」「Probe 対象」「解析式」をデモ定義 JSON に書く
  (シェルがレンダリング)。デモ固有コードを最小化する。

## 3. 回路エディタ(D19 電気工作台)

- **配線モデル**: 回路はノードグラフ(部品 = 2 端子/多端子素子、配線 = 理想導体ノード結合)。
  内部表現は `CircuitDesc`(ノード ID + 素子リスト、[13-electromagnetism/02](../13-electromagnetism/02-circuits.md) の
  MNA 入力)と 1:1 対応 — エディタ操作は `CircuitDesc` の編集であり、World の再構築
  (create_circuit)として適用する。
- **UI**: 2D 盤面(ブレッドボード風グリッド)。部品パレット(電池・抵抗・LED・コンデンサ・
  コイル・スイッチ・モーター端子・計器)からドラッグ配置、端子間をクリックで配線。
  配線はノード自動割当(接触した配線同士は同一ノードへ結合)。
- **計器**: 電圧計(`circuit_probe`)・電流計(素子の電流クエリ)は Probe として配置し、
  シェルのグラフに接続(§1 機能 2)。
- **検証との接続**: エディタで組んだ回路がそのまま E3/E4/E5 の検証シーンになる
  (プリセット読込 = シーン JSON)。短絡・過電流はヒューズイベント(`FuseBlown`)で表現し、
  ジュール熱は熱ドメインへ(D19 の合格基準)。
- 3D 世界との接続: 盤面はシーン内の「工作台」オブジェクトに対応。モーター端子・
  導体棒等の電磁⇔力学素子([13-electromagnetism/05](../13-electromagnetism/05-em-mechanics-coupling.md))は
  3D 側の実体とリンク表示する。

## 4. シーン編集・スポーン

- **スポーンパレット**: 形状(球・箱・カプセル)× 材質(MaterialDb のプリセット)を選んで
  クリック配置(`create_body`)。「密度だけ変えた木」等の材料派生
  (`extends`、[20-integration/04](../20-integration/04-world-api.md) §3)を UI で作れる。
- **つかむ・投げる**: マウスドラッグ = `Grab`/`MoveGrab`/`Release` コマンド(ソフト拘束)。
  実行中の介入はすべてコマンドなのでリプレイに記録される。
- **編集モード**: 一時停止中のみ位置・パラメータの直接編集を許す(編集 = シーン JSON の
  変更として記録し、「編集後の状態」を新しいシナリオとして保存できる)。実行中の直接編集は
  不可(決定論とリプレイの一貫性 — すべてコマンド経由)。
- **保存・共有**: シーン JSON + リプレイのエクスポート/インポート。数値は f64 を損なわない
  表現([20-integration/02](../20-integration/02-determinism-replay.md) §4.1)。

## 5. ヘッドレス CI テスト基盤

- デモ合格基準(D1–D43)は「シーン JSON + 入力列(スクリプト)+ Probe assert(期待値・
  許容誤差)」のヘッドレステストとして記述し、ネイティブ(`cargo test`)と wasm(node)で
  実行する。目視チェックは別途(合格 = ヘッドレス Green + 目視)。
- シェルの UI コンポーネント自体のテストは最小(スモーク)に留める — 検証の重心は
  物理側(World API 経由)にあり、UI は薄い写像を保つ。

## 6. 実装フェーズ対応

- Phase 0: Vite + Three.js 雛形、wasm 疎通(既存計画どおり)。
- Phase B 各ウェーブ: シェル(6 機能)を math ウェーブ後に最小実装 → 各ウェーブの
  スモークデモで随時拡充。予測→実験(§2)は D1 で最初に成立させる。
- P4: 回路エディタ(D19)。Phase C: シーン編集・共有の完成、全デモのヘッドレス CI。

## 7. 検証

- シェル 6 機能が World API のみで実装されている(特権アクセス不在 —
  [20-integration/04](../20-integration/04-world-api.md) §5 と同じ検証)。
- 予測→実験遷移の E2E スモーク(D1 で予測入力→比較表示まで)。
- 回路エディタ: 組んだ `CircuitDesc` の MNA 解が回路解析の手計算ケースと一致(E5 と同基準)。
- リプレイ保存→読込→再生のハッシュ一致(シェル経由で階層 1 を維持)。
