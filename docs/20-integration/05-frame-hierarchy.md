# 横断 05. フレーム階層 — floating origin の詳細設計

crate: `sim-core` / `sim-world`。位置表現の決定(C-3、
[00-foundation/02-scale-ladder.md](../00-foundation/02-scale-ladder.md) §2.2:
剛体の位置は所属フレームのローカル座標 f64、World はフレーム ID + ローカル座標を正準状態)を
実装可能な水準まで詳細化する。

## 1. 目的と位置づけ

- 空間上限を太陽系 ~$10^{12}$ m に広げても、地表スケールの物理(mm〜m 精度)を
  **原点近傍の f64 高精度領域**で解けるようにする。
- フレーム階層は**コアで予約された機構**であり、天体スケール(L5)はその具体化である
  ([00-foundation/04-architecture.md](../00-foundation/04-architecture.md) §4)。
  主戦場シーン(地表のみ)ではルートフレーム 1 個で動き、コストはフレーム索引 1 個のみ。

## 2. フレームの定義

```rust
pub struct Frame {
    pub id: FrameId,                  // ROOT = 0。世代付き ID
    pub parent: Option<FrameId>,      // ROOT のみ None。木構造(閉路なし、ロード時検証)
    /// 親フレーム内でのこのフレームの原点位置・姿勢・速度・角速度。
    /// 天体フレームでは L5(軌道力学)が毎(天体)ステップ供給する。静的フレームでは定数。
    pub origin_in_parent: Vec3,
    pub rotation_in_parent: Quat,
    pub velocity_in_parent: Vec3,
    pub angular_velocity_in_parent: Vec3,
    /// このフレームの一様重力(地表フレーム)または重力源(天体フレーム)
    pub gravity: FrameGravity,
}
```

- 階層の代表例: `ROOT(系重心)→ 天体中心(地球)→ 地表ローカル`。深さは実用上 ≤ 4。
- フレーム変換 $T_{A\to B}$ は親子鎖をたどる合成で得る(共通祖先まで上がって下る)。
  変換の合成順序は経路で一意(木構造なので分岐なし)。
- **ソルバはフレームを知らない**: 剛体・流体・接触のすべての演算は同一フレーム内で閉じる
  ([00-foundation/04](../00-foundation/04-architecture.md) §3)。フレームを扱うのは
  World のフレーム管理(本文書)と L5 だけである。

## 3. 跨ぎ判定(re-parenting)

物体がフレームの管轄領域を出入りしたら所属フレームを付け替える:

- **管轄領域**: 各フレームは半径 $r_{frame}$ の球(または天体の重力圏 SOI)を管轄とする。
- **ヒステリシス付き閾値**: 出るときは $r > 1.1\,r_{frame}$、入るときは $r < 0.9\,r_{frame}$。
  境界での毎ステップ往復(チャタリング)を防ぐ。
- 判定は**ステップ末尾のイベント確定段**([00-foundation/04](../00-foundation/04-architecture.md)
  §1.4.3 の段 7)で行い、付け替えは**次ステップ先頭**に適用する(ステップ途中で座標系が
  変わらない — ソルバ不変条件)。
- 付け替え時の状態変換: $\mathbf{x}' = T_{A\to B}\,\mathbf{x}$、
  $\mathbf{v}' = R_{A\to B}(\mathbf{v} - \mathbf{v}_{B\ in\ A}) - \boldsymbol\omega_{B}\times(\mathbf{x}' )$
  (位置・速度・姿勢・角速度を厳密変換。運動量・エネルギーはフレーム相対量として再計算し、
  台帳に「フレーム跨ぎ」として記帳 — §5)。

## 4. 跨ぎ時の接触・拘束の扱い

- **接触**: 接触は同一フレーム内のペアにのみ発生させる(broadphase がフレームで分割)。
  跨ぎ直後のステップでは、旧フレームの接触マニフォールドを新フレームへ**座標変換して持続**
  させる(ウォームスタートを失わない)。異フレーム間の物体は接触しない —
  近接しうる場合は先に同一フレームへ寄せる(跨ぎ判定の管轄領域はこれが起きないよう
  重なりを持たせて設計する)。
- **拘束(ジョイント)**: ジョイントで結ばれた剛体集合(アイランド)は**常に同一フレーム**に
  属する(不変条件、validator が強制)。跨ぎ判定はアイランド単位で行い、代表点
  (質量重心)で判定して全体を一括付け替えする。
- **流体・場**: 格子・SPH 領域はフレームに固定(領域ごと跨ぎはしない)。流体領域と剛体の
  結合は同一フレーム内のみ。

## 5. 非慣性項の台帳計上

回転・加速するフレーム(地表ローカル: 自転、天体: 公転)内の物体には見かけの力が働く:

- 遠心力 $\mathbf{F}_{cf} = -m\,\boldsymbol\omega\times(\boldsymbol\omega\times\mathbf{r})$、
  コリオリ力 $\mathbf{F}_{co} = -2m\,\boldsymbol\omega\times\mathbf{v}$、
  オイラー力 $\mathbf{F}_{eu} = -m\,\dot{\boldsymbol\omega}\times\mathbf{r}$。
  フレームの $\boldsymbol\omega$ が閾値未満(既定: 地球自転 $7.3\times10^{-5}$ rad/s 未満)の
  シーンでは**既定オフ**(近似バッジに表示)、天体シーン・フーコー振り子等ではオン。
- **台帳計上**: 非慣性項は保存力ではないため、その仕事
  $W_{ni} = \sum \mathbf{F}_{ni}\cdot\mathbf{v}\,\Delta t$ を台帳の独立項 `frame_inertial` に
  記帳する(コリオリ力は $\mathbf{v}\perp\mathbf{F}$ で仕事ゼロ — 検算に使う)。
  フレーム跨ぎ時の運動エネルギー差(フレーム相対速度の変化分)も同項に記帳し、
  台帳残差([00-foundation/04](../00-foundation/04-architecture.md) §1.1.2(2))に
  漏らさない。

## 6. 跨ぎの決定論

- 跨ぎ判定は状態の決定的関数(ヒステリシス付き閾値、§3)であり、壁時計・描画に依存しない。
- 同一ステップに複数の跨ぎイベントが発生した場合、$(step, \text{アイランド代表 ID}, \text{方向})$
  の辞書式順で**全順序化**して適用する([20-integration/02](02-determinism-replay.md) §2 の
  イベント順序規約と同型)。
- 跨ぎイベントはリプレイのチェックポイント対象(跨ぎ step とフレーム ID を記録)。
  状態ハッシュはフレーム ID + ローカル座標をそのままハッシュする(グローバル座標に
  変換しない — 正準状態が正)。

## 7. 単体テスト(跨ぎ)

| テスト | 基準 |
|---|---|
| 往復変換 | $T_{B\to A}\circ T_{A\to B} = \mathrm{id}$(abs 1e-12) |
| 跨ぎの物理不変 | 自由運動中の物体がフレームを跨いでも、ROOT 換算の軌道が跨ぎなしの参照解と一致(rel 1e-9) |
| チャタリング防止 | 境界に静止した物体が 1000 step で跨ぎイベント ≤ 1 回 |
| ウォームスタート持続 | 接触したまま跨ぐ箱スタックが跨ぎ後も静止(M12 と同基準) |
| 決定論 | 跨ぎを含むシーンの 2 回実行ハッシュ一致 + スナップショット再開一致(階層 1) |
| コリオリ検算 | 回転フレーム内の自由粒子の軌道 = 慣性系解の座標変換(rel 1e-6)、コリオリ仕事 = 0(abs 1e-12) |

## 8. 実装フェーズ対応

- P1(力学基礎): `FrameId` を剛体状態に導入(全て ROOT の単一フレーム動作)。
  型・ハッシュ・スナップショットにフレームを含める。
- Pα(天体): 複数フレーム・跨ぎ判定・非慣性項・L5 供給の実装、§7 の跨ぎテスト Green。
