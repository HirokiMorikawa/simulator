# 横断 06. レジーム切替 — 時間加速と天体独立時間軸のプロトコル

crate: `sim-world` / `sim-astro`。固定 dt 規約と時間加速(D34 太陽系儀の年単位、
D37 再突入の実時間)を両立させる切替プロトコル。
方針は [00-foundation/02-scale-ladder.md](../00-foundation/02-scale-ladder.md) §2.3。

## 1. 時間加速の 2 レジーム(固定 dt を破らない)

World の $\Delta t$(1/120 s)は**どのレジームでも変更しない**。時間加速は次の 2 方式で実現する:

- **通常レンジ(〜128 倍)**: 「1 描画フレームに複数物理ステップ」。加速率 $k$ 倍 =
  1 フレームに $k \times \Delta t / (1/60)$ ステップ実行。$\Delta t$ 固定のまま
  物理時間だけが速く進む。上限は性能予算(シーンクラス、
  [00-foundation/05](../00-foundation/05-rust-wasm-platform.md) §5.1)から決まり、
  達しない場合は実効加速率を UI に表示(静かに遅くしない)。
- **天体レンジ(D34: 日〜年/秒)**: 局所物理(剛体・流体等)を持たない**天体専用レジーム**に
  切り替え、天体状態のみを**独立時間軸**(シンプレクティック積分、刻み ~0.1 日 =
  [16-astro/01](../16-astro/01-gravitation-nbody.md) §4.2)で進める。World 時刻は天体時刻に
  従属し、ローカル物理ソルバは停止(状態は凍結保存)。

どちらも**加速率の変更はコマンド**(`SetTimeScale`)として入力列に記録され
([20-integration/02](02-determinism-replay.md) §4.2)、リプレイで同一に再生される。

## 2. レジームの定義

```rust
pub enum TimeRegime {
    /// ローカル物理 + (必要なら)天体を World dt に従属させて共進行
    Local { steps_per_frame: u32 },
    /// 天体のみ。独立時間軸 dt_astro(固定、既定 0.1 日)。ローカル物理は凍結
    Astro { dt_astro: f64, steps_per_frame: u32 },
}
```

- レジームは World の状態の一部(スナップショット・ハッシュ・リプレイに含まれる)。
- D37 再突入のような「両時間軸が意味を持つ遷移」は、**切替イベントの前後で一方だけが
  生きている**ように設計する(同時に 2 軸を進めない — 決定論と台帳の単純性を守る)。

## 3. 切替プロトコル

### 3.1 切替時刻の量子化

- 切替(Local → Astro、Astro → Local)は **World ステップ境界でのみ**発生する。
  Astro レジーム中は天体ステップ境界(= World 時刻が $dt_{astro}$ 刻みで進む)でのみ発生する。
- 切替条件(高度・動圧・推力などの閾値、または明示コマンド)は**状態の決定的関数**として
  ステップ末尾に評価し、次ステップ先頭で切り替える
  ([02-determinism-replay.md](02-determinism-replay.md) §2 の適応的アルゴリズム規約)。
  閾値にはヒステリシスを付ける(フレーム跨ぎ判定
  [05-frame-hierarchy.md](05-frame-hierarchy.md) §3 と同型)。

### 3.2 状態受け渡し

- **Astro → Local**(例: 再突入開始): 天体状態(軌道位置・速度)→ ローカル初期条件の写像。
  カプセルの軌道状態を突入天体の地表(回転)フレームのローカル座標・速度に厳密変換し
  ([05-frame-hierarchy.md](05-frame-hierarchy.md) §3 の変換式)、凍結していたローカル物理を
  この初期条件で再開する。大気・重力はフレームの環境として設定される。
- **Local → Astro**(例: 軌道投入完了): ローカル状態の質量重心・速度を天体状態へ集約し、
  ローカル詳細(姿勢・内部状態)はスナップショットとして保存(再突入時の復元に使える)。
- 受け渡しはエネルギー・運動量をフレーム変換として厳密に保ち、変換差分は台帳の
  `frame_inertial` 項に記帳([05-frame-hierarchy.md](05-frame-hierarchy.md) §5)。

### 3.3 巻き戻し可否

- スナップショット([00-foundation/04](../00-foundation/04-architecture.md) §5)はレジームを
  含むため、**切替を跨ぐ巻き戻しは可能**(切替前の面に戻れば旧レジームごと復元される)。
- Astro レジーム中のスナップショット間隔は天体時間基準(既定: 表示 1 s ごと)に切り替え、
  リングバッファ面数は共通(メモリ予算も共通)。
- 巻き戻し後の再実行は、記録済み入力列(加速率変更・切替コマンド込み)を再適用すれば
  ビット一致で再現される(階層 1、[02-determinism-replay.md](02-determinism-replay.md) §5)。

## 4. 検証

- **切替を跨ぐリプレイ一致**(D34/D37 の合格基準に追加、
  [21-verification/03](../21-verification/03-demo-scenarios.md)): 加速→減速・
  Astro→Local→Astro を含むセッションの記録リプレイが全チェックポイントハッシュ一致。
- 切替時刻の量子化: 切替イベントの発生 step が入力列から一意に決まる(2 回実行で同一 step)。
- 状態受け渡しの保存性: Astro→Local 変換前後で ROOT 換算のエネルギー・運動量が一致
  (rel 1e-9、大気・重力モデルの差し替え分は台帳に明示)。
- 巻き戻し: 切替 step を跨いで 2 面前に戻し再実行 → 連続実行とハッシュ一致。

## 6. パラメータ表

| パラメータ | 既定値 | 根拠 |
|---|---|---|
| 通常レンジ加速上限 | 128 倍 | 1 フレーム 256 step(dt=1/120、60 fps)が性能予算の上限目安 |
| $dt_{astro}$ | 0.1 日 | 内惑星軌道の位相誤差が A1 許容(0.1%)に収まる刻み([16-astro/01](../16-astro/01-gravitation-nbody.md) §4.2) |
| 切替ヒステリシス | 条件依存(例: 再突入 高度 120/140 km) | チャタリング防止([05](05-frame-hierarchy.md) §3 と同型) |
