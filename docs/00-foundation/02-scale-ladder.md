# 02. スケールの階梯と有効理論 — 本エンジンの根本思想

## 1. 問題: 「すべてを量子力学で解く」ことはできない

現実世界の土台は量子力学である。ならば量子力学だけを実装すれば現実の写像が得られるはずだが、これは**原理的に不可能**である。定量的に示す。

- 水 1 g には約 $3.3\times10^{22}$ 個の分子(アボガドロ数 $N_A = 6.022\times10^{23}\,\mathrm{mol^{-1}}$、出典: CODATA 2018)が含まれる。
- $N$ 粒子系の波動関数は $3N$ 次元の配置空間上の複素関数である。1 次元あたりわずか 10 点で離散化しても、
  必要なメモリは $10^{3N}$ 複素数 — $N=100$ ですら $10^{300}$ 個であり、観測可能な宇宙の原子数($\sim10^{80}$)を超える。
- つまり「量子力学の直接計算」は**数十粒子**が実用上界である(現代のスパコンでも厳密対角化は 50 スピン程度)。

これはハードウェアの進歩で解決する問題ではなく、状態空間の指数爆発という数学的事実である。
したがって「物理の全域を押さえた現実の写像」の唯一の実現可能な形は、
**現実の科学・工学がやっているのと同じこと** — スケールごとに最良の**有効理論 (effective theory)** を使い、
理論間の受け渡しを明示的に設計すること — である。

有効理論とは「あるスケール以下の自由度を平均化(粗視化)して得られる、そのスケールで正確な理論」を指す。
熱力学は分子運動の、流体力学は分子集団の、剛体力学は弾性体の粗視化である。
粗視化は情報の劣化ではなく、**そのスケールで意味のある変数だけを残す**操作であり、
各有効理論はその適用域内で検証済みの精度を持つ。

## 2. スケールの階梯 (scale ladder)

本エンジンが扱う理論を、空間・時間スケールと自由度の粗視化の順に並べる。

| 階層 | 理論 | 代表スケール | 状態変数 | 本エンジンでの実装形態 |
|---|---|---|---|---|
| L0 | 量子力学 | $10^{-10}$–$10^{-8}$ m, $10^{-16}$–$10^{-12}$ s | 波動関数 $\psi$ | 少数自由度の直接解(専用シナリオ)+ 上位層への物性供給 |
| L1 | 統計力学・分子運動論 | $10^{-9}$–$10^{-6}$ m | 分布関数 $f(\mathbf{x},\mathbf{v},t)$、アンサンブル | 粒子系デモ(気体・イジング)+ 輸送係数・状態方程式の根拠 |
| L2 | 連続体力学(流体・熱・弾性) | $10^{-6}$–$10^{2}$ m | 場: $\rho, \mathbf{u}, T, p$ | 格子/粒子ソルバ(Navier-Stokes、熱伝導) |
| L3 | 剛体・多体力学 | $10^{-2}$–$10^{2}$ m | 剛体状態 $(\mathbf{x}, q, \mathbf{v}, \boldsymbol{\omega})$ | 剛体ソルバ + 拘束(本エンジンの主戦場) |
| L4 | エンティティ層 | $10^{-1}$–$10^{2}$ m | 関節骨格・制御状態 | 物・人・乗り物・生物(物理層のクライアント) |
| L5 | 天体力学 | $10^{6}$–$10^{12}$ m, $10^{3}$–$10^{9}$ s | 点質量 $(\mathbf{x}, \mathbf{v}, m)$、軌道要素 | N 体重力(Barnes-Hut)・軌道・再突入([16-astro/](../16-astro/))。相対論オプトイン |
| 横断 | 電磁気学 | 全域 | 場 $\mathbf{E}, \mathbf{B}$、回路変数 | 回路(集中定数)・準静的場・FDTD・幾何光学の使い分け |

**設計原則: 各階層は自分の階層の方程式を解き、下の階層からはパラメータと法則を受け取り、隣接階層とは結合条件でつながる。**

L3(剛体)と L5(天体)は同じニュートン力学だが、スケール・積分器・データ構造が異なる:
天体は点質量 + シンプレクティック積分 + 独立時間軸。地表の剛体物理と天体軌道は
**floating origin フレーム**(§2.3)でつなぐ。

### 2.1 階層間の受け渡しの実例

下から上へ(物性・法則の供給):

- L0→L2/L3: 原子間結合 → ヤング率・密度・比熱・熱伝導率・摩擦係数などの**材料物性**。
  本エンジンでは第一原理計算はせず、実測物性表([12-thermal/04-material-thermal-props.md](../12-thermal/04-material-thermal-props.md) 等)を使う。
  「なぜその値なのか」の説明責任を L0 文書([14-quantum/03-effective-models.md](../14-quantum/03-effective-models.md))が担う。
- L0→横断: バンド構造 → 半導体素子の I-V 特性、光の吸収・発光スペクトル。
- L1→L2: 分子運動論 → 粘性係数 $\mu$、拡散係数 $D$、理想気体の状態方程式 $pV=Nk_BT$。
  マクスウェル=ボルツマン分布のシミュレーション([15-statistical/02-kinetic-gas.md](../15-statistical/02-kinetic-gas.md))は
  この導出を**ユーザーが目で確かめられる**ようにするためのものである。
- L1→L3: 揺動散逸定理 → ブラウン運動(微小粒子に働くランダム力の大きさは粘性抵抗と温度から決まる)。

同階層・上下の結合(双方向の力・エネルギー交換):

- L2⇔L3: 流体が剛体に浮力・抗力を与え、剛体が流体を押しのける([11-fluid/05-aero-hydrodynamics.md](../11-fluid/05-aero-hydrodynamics.md))。
- L3→L2(熱): 摩擦仕事 → 発熱(エネルギー保存の橋、[20-integration/01-coupling-matrix.md](../20-integration/01-coupling-matrix.md))。
- 横断⇔L3: ローレンツ力、モーターのトルク([13-electromagnetism/05-em-mechanics-coupling.md](../13-electromagnetism/05-em-mechanics-coupling.md))。
- L5→L3: 天体重力の局所場化(一様重力 $g$ は N 体重力の局所近似)、潮汐力、太陽輻射
  ([16-astro/01-gravitation-nbody.md](../16-astro/01-gravitation-nbody.md))。
- L5⇔L2/熱: 大気圏再突入(高度依存大気の抗力 + 空力加熱、[16-astro/02-orbital-mechanics.md](../16-astro/02-orbital-mechanics.md))。

### 2.2 座標系の階梯と floating origin(L5 拡張に伴う設計)

空間上限を太陽系 ~$10^{12}$ m に広げると、単一の絶対座標では地表スケールの精度が問題になりうる。

- **f64 絶対座標の分解能**: $2^{-52} \approx 2.2\times10^{-16}$ の相対精度。原点を太陽(または系重心)に置くと、
  Neptune 距離($4.5\times10^{12}$ m)での位置分解能は $\approx 1$ mm。**軌道力学には十分**
  (惑星の直径・軌道に対して桁違いに小さい)。
- **表面スケール物理では floating origin に切替**: 遠方天体の地表で剛体・流体(mm〜m 精度が要る)を
  回すときは、その天体を原点とする**浮動原点フレーム**でローカルに計算し、天体の絶対位置は
  L5 で別管理する。これによりローカル物理は原点近傍の高精度領域で解ける。
- **位置表現の決定**: 剛体の位置は**所属フレームのローカル座標 f64**で
  保持し、World は**フレーム ID + ローカル座標**を正準状態とする
  ([00-foundation/04](04-architecture.md) §1.1.2(1)/§3、
  [10-mechanics/01](../10-mechanics/01-rigid-body.md) §3 の型定義に反映)。
  floating origin を後付けの座標シフトではなく最初からデータ設計に織り込む。主戦場シーン
  (地表のみ)ではルートフレーム 1 個で従来どおり動作し、コストはフレーム索引 1 個のみ。
- **フレーム階層の詳細設計は専用文書が正**:
  [20-integration/05-frame-hierarchy.md](../20-integration/05-frame-hierarchy.md) が、
  階層の定義(系重心 → 天体中心 → 地表ローカル)、跨ぎ判定(ヒステリシス)、跨ぎ時の
  接触・拘束、非慣性項(コリオリ・遠心)の台帳計上、跨ぎの決定論と単体テストを規定する。

### 2.3 時間スケールの分離

天体(軌道周期 ~$10^{8}$ s)と剛体($\Delta t = 1/120$ s)は $10^{10}$ 倍のスケール差があり、
同一刻みで co-step するのは無意味。

- 天体は**独立時間軸**(自分の大刻み、[16-astro/01](../16-astro/01-gravitation-nbody.md) §4.2)で進める。
- 近接イベント(大気圏再突入・ロケットのバーン・接近通過)では、L5 が**決定論的にレジームを
  微細刻みへ切替**える。切替条件(高度・動圧・推力)は状態から決定的に判定(適応刻みの
  非決定性を排除、[20-integration/02-determinism-replay.md](../20-integration/02-determinism-replay.md))。
- **切替プロトコルは専用文書が正**:
  [20-integration/06-regime-switching.md](../20-integration/06-regime-switching.md) が、
  時間加速の 2 レジーム(フレーム当たり複数ステップ / 天体独立時間軸)、切替時刻の量子化
  (World ステップ境界のみ)、状態受け渡し(天体状態 ⇔ ローカル初期条件)、巻き戻し可否を
  規定する。D34/D37 の合格基準に「切替を跨ぐリプレイ一致」を含む。
- FDTD・量子([13-electromagnetism/03](../13-electromagnetism/03-maxwell-fdtd.md),
  [14-quantum/02](../14-quantum/02-schrodinger-solver.md))と同じ「独立時間軸の専用ソルバ」パターン。

## 3. 理論選択の判定基準

「この現象はどの理論で解くべきか」の判定を、無次元数と限界スケールで明文化する。各ドメイン文書の第 5 節がこの表を詳細化する。

| 判定基準 | 閾値 | 下側の理論 | 上側の理論 |
|---|---|---|---|
| クヌーセン数 $Kn = \lambda/L$ (平均自由行程/代表長) | $Kn < 0.01$ | 連続体(Navier-Stokes) | 分子運動論(希薄気体) |
| 熱的ド・ブロイ波長 $\lambda_{th} = h/\sqrt{2\pi m k_B T}$ vs 粒子間距離 | $\lambda_{th} \ll d$ | 古典統計 | 量子統計 |
| 系のサイズ vs 電磁波長 $\lambda = c/f$ | $L \ll \lambda$ | 回路(集中定数)・準静的場 | 波動(FDTD) |
| 物体の変形量 vs サイズ | 変形 $\ll$ サイズ | 剛体 | 弾性体・ソフトボディ |
| 作用 vs プランク定数 $S/\hbar$ | $S \gg \hbar$ | 古典力学 | 量子力学 |
| 速度 vs 音速(マッハ数 $Ma$) | $Ma < 0.3$ | 非圧縮流体 | 圧縮性流体(当面対象外) |
| 重力ポテンシャル $GM/(rc^2)$・$v/c$ | $\ll 10^{-6}$ | ニュートン力学(既定) | 1PN 相対論補正(天体のみオプトイン、[16-astro/03](../16-astro/03-relativistic-corrections.md)) |
| 物体サイズ vs 原点距離 | 高精度が要る表面物理 | 単一絶対座標 | floating origin フレーム(§2.2) |

具体例: 空気の平均自由行程は $\lambda \approx 68\,\mathrm{nm}$(標準状態、出典: Sone, *Molecular Gas Dynamics*)。
日常物体($L > 1\,\mathrm{mm}$)では $Kn < 10^{-4}$ で連続体近似は極めて良い。
一方、室温の電子の $\lambda_{th} \approx 4.3\,\mathrm{nm}$ は原子スケールに達し、電子の振る舞いは量子で扱う必要がある —
これが「材料物性は量子由来、物体の運動は古典」という分業の根拠である。

## 4. アーキテクチャへの帰結

この章の結論がそのまま [04-architecture.md](04-architecture.md) の構造になる:

1. **ドメイン = Solver モジュール**: 各有効理論を独立のソルバとして実装する。
   各ソルバは自分の支配方程式・状態・数値解法・適用域を持ち、他ソルバの内部に依存しない。
2. **結合 = Coupling 層**: 理論間の受け渡し(力・熱・物性・境界条件)をソルバの外に明示的に置く。
   結合の全体像は結合行列([20-integration/01-coupling-matrix.md](../20-integration/01-coupling-matrix.md))で一望できる。
3. **オーケストレータ**: タイムスケールが桁違いのソルバ群(剛体 $\Delta t \sim 10^{-2}$ s、FDTD $\Delta t \sim 10^{-11}$ s)を
   operator splitting と sub-stepping で束ねる。
4. **物性は共有データベース**: 材料物性(密度・比熱・摩擦係数・誘電率…)は全ドメインが参照する単一のテーブルとし、
   出典を管理する。

## 5. この思想がユーザー体験に与えるもの

- 「なぜ木は浮いて石は沈むのか」を、浮力(L2)→ 密度(L0 由来の物性)へと**階層を降りて説明できる**。
  デモシナリオは可能な限り階層間のつながりを見せる構成にする(例: 気体分子デモの圧力測定値が、
  連続体の状態方程式と一致することを並べて表示する)。
- どの現象がどの近似で計算されているかを UI から確認できる(「このシーンでは空気は一様媒質近似」等)。
  忠実さの限界を隠さないことは、検証して遊ぶ体験の一部である。
