# 04. 乱数 — 決定論的 PRNG と分布サンプリング

統計力学(熱ゆらぎ・ブラウン運動・モンテカルロ)・量子測定・シーン生成が使う乱数基盤。
**決定論**([20-integration/02-determinism-replay.md](../20-integration/02-determinism-replay.md))の要件から、
すべての乱数はここで定義する PRNG のみを使用する。`rand` crate・OS エントロピーはコアで禁止。

## 1. PRNG: PCG32 を採用

```rust
/// PCG-XSH-RR 64/32 (O'Neill 2014)。状態128bit、周期 2^64、ストリーム分割可。
pub struct SimRng { state: u64, inc: u64 }

impl SimRng {
    pub fn new(seed: u64, stream: u64) -> SimRng;
    pub fn next_u32(&mut self) -> u32;
    pub fn next_f64(&mut self) -> f64;            // [0,1)。53bit精度: (next_u64 >> 11) * 2^-53
    pub fn range_f64(&mut self, lo: f64, hi: f64) -> f64;
    pub fn range_u32(&mut self, n: u32) -> u32;   // [0,n)。棄却法でモジュロバイアス除去
}
```

選定理由: (1) 実装が ~10 行で透明、(2) 統計品質が TestU01 BigCrush 通過水準、(3) **ストリーム機能**
(`inc` の違いで独立系列)により、ドメインごと・粒子ごとの独立乱数列を単一シードから決定的に導出できる。
mulberry32 より状態空間・品質に余裕があるため PCG を選ぶ。

## 2. ストリーム配分

World のマスターシード $s$ から、用途別ストリームを固定規約で導出する:

```
SimRng::new(s, STREAM_SCENE)        // シーン初期配置
SimRng::new(s, STREAM_STATISTICAL)  // 分子初期速度・ランジュバンノイズ
SimRng::new(s, STREAM_QUANTUM)      // 測定の確率的選択
SimRng::new(s, STREAM_DIAG)         // テスト・診断 (物理に影響しない)
```

- ソルバ間で RNG を共有しない(呼び出し順の変化が他ドメインの結果を変えるのを防ぐ)。
- 並列化時は粒子チャンクごとに `stream = base + chunk_id` で分割し、スレッドスケジューリングと
  無関係に系列を固定する。

## 3. 分布サンプリング

| 分布 | 方法 | 用途 |
|---|---|---|
| 一様 $[0,1)$ | §1 | 基本 |
| 正規 $N(0,1)$ | Ziggurat ではなく **Box-Muller(対数版)** | ランジュバンノイズ、マクスウェル分布初期化。Ziggurat は表引きの実装差リスクがあるため、遅くても単純な Box-Muller を採用(ホットパスではない) |
| マクスウェル=ボルツマン速度 | 各成分独立に $N(0, \sqrt{k_BT/m})$ | 気体初期化 [15-statistical/02](../15-statistical/02-kinetic-gas.md) |
| 単位球面一様 | Marsaglia 法(棄却) | 方向サンプリング |
| 指数分布 | 逆関数法 $-\ln(1-u)/\lambda$ | 衝突時間・放射 |
| 離散分布 | 累積和 + 二分探索 | 量子測定の確率的選択 [14-quantum/02](../14-quantum/02-schrodinger-solver.md) |

Box-Muller は 2 値生成のためキャリー(残り 1 値)を持つ — キャリーも状態の一部として
スナップショット・ハッシュに含める(決定論)。

## 4. 決定論との関係(規約)

1. 乱数消費量はコードパスに対して固定: 「条件付きで乱数を引く」箇所では、分岐の両側で消費数を揃えるか、
   ストリームを分ける。
2. `SimRng` の全状態はスナップショットに含める(リプレイ再開で系列が続く)。
3. シード 0 も有効値として扱う(PCG は状態初期化で撹拌するため問題ない)。
4. 「乱数を変えても物理法則は同じ」— シードは初期条件・ゆらぎの実現値のみを変える。
   これがユーザーの遊び方(同条件で別の偶然を見る / 完全再現する)の基盤になる。

## 5. 検証

- 参照値テスト: PCG 公式テストベクタとの一致(実装の正しさ)。
- 統計テスト(CI では軽量版): 一様性のカイ二乗、正規サンプルのモーメント($10^6$ 点で
  平均 $<3\sigma/\sqrt{N}$、分散相対誤差 <1%)。
- 決定論: 同一 (seed, stream) → 同一系列(ビット一致)、異なる stream の系列間相関が無視できること。
