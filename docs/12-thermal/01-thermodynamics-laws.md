# 熱 01. 熱力学法則 — 温度・状態方程式・エネルギー収支の設計

crate: `sim-thermal`。熱ドメインの原理と、エンジン全体のエネルギー会計の設計。

## 1. 担う現実の現象

物の熱さ・冷たさ、暖まる・冷める、気体の圧縮で熱くなる、エンジン・冷蔵庫の効率限界。
遊び方の例: ピストンで気体を断熱圧縮して温度上昇を見る、カルノーサイクルを組んで
効率が $1 - T_c/T_h$ を超えられないことを確かめる。

## 2. 支配方程式

### 2.1 熱力学第 0〜3 法則(エンジンでの意味)

- **第 0 法則**(温度の存在): 各熱容量ノードは単一の温度 $T$ を持つ。接触で熱平衡へ向かう。
- **第 1 法則**(エネルギー保存): $dU = \delta Q - \delta W$。
  **エンジン全体のエネルギー会計の根拠**(§4)。あらゆる散逸(摩擦・非弾性衝突・抗力・
  ジュール熱)は消えるのではなく内部エネルギー(温度)に変わる。
- **第 2 法則**(エントロピー増大): 孤立系で $dS \ge 0$。熱は自発的に高温→低温にのみ流れる
  (伝導則がこれを自動で満たす)。ヒートポンプ・エンジンの効率限界。
  エントロピーは観測量として計算・表示する(§4.2)。
- **第 3 法則**: $T \to 0$ の極限規定。実用上は $T > 0$ のアサーションと低温物性の限界表示のみ。

### 2.2 状態方程式(気体)

理想気体: $pV = nRT$、内部エネルギー $U = \frac{f}{2}nRT$(自由度 $f$: 単原子 3、二原子 5)。
比熱 $c_v = \frac{f}{2}R/M$、$c_p = c_v + R/M$、比熱比 $\gamma = c_p/c_v$。
断熱過程: $pV^\gamma = const$、$TV^{\gamma-1} = const$。
実在気体(van der Waals)は表示・比較用オプション(相転移の定性議論、Phase 5)。

### 2.3 集中熱容量ノード(本エンジンの基本モデル)

物体内温度が一様とみなせる条件(ビオ数 $Bi = hL/k < 0.1$)で、物体 1 個 = 熱ノード 1 個:

$$C\,\frac{dT}{dt} = \sum_i \dot{Q}_i \qquad (C = mc_p)$$

$\dot{Q}_i$: 伝導・対流・放射・摩擦熱・ジュール熱などの熱流([02-heat-transfer.md](02-heat-transfer.md))。
$Bi \ge 0.1$(大きい・熱伝導の悪い物体)は内部分割(形状のボクセル化 or 殻分割)で
多ノード化する(Phase 3)。

## 3. 状態表現・Rust 型定義

```rust
pub struct ThermalNode {
    pub temperature: f64,          // [K]
    pub heat_capacity: f64,        // C = m·c_p [J/K] (質量・材料から自動)
    pub emissivity: f64,           // 放射用 [0,1]
    pub area: f64,                 // 表面積 (形状から自動)
    pub heat_accum: f64,           // ステップ内の熱流入 [J]
}

/// 気体区画 (閉じた容器・ピストン)
pub struct GasCompartment {
    pub n_moles: f64,
    pub volume: f64,               // 機構 (ピストン位置) から従属
    pub temperature: f64,
    pub gas: GasSpecies,           // f, M (Air: f=5, M=28.97 g/mol)
}
impl GasCompartment {
    pub fn pressure(&self) -> f64 { self.n_moles * R * self.temperature / self.volume }
}
```

- 剛体 1 個につき ThermalNode 1 個を既定で付与(温度は RigidBodySet と共有、
  [10-mechanics/01](../10-mechanics/01-rigid-body.md) §3)。
- 環境(大気・水域)は「無限熱容量ノード」(温度固定)または有限容量(部屋の暖まり)を選べる。

## 4. エネルギー会計(全ドメイン横断の設計)

**エネルギー台帳 (EnergyLedger)** をエンジンの第一級機能とする:

```rust
pub struct EnergyLedger {
    pub kinetic: f64, pub potential: f64, pub elastic: f64,
    pub thermal: f64,              // Σ C·(T − T_ref)
    pub em_field: f64, pub chemical_battery: f64,
    pub injected: f64,             // モーター・ユーザー操作による注入 (符号つき)
    pub numerical_residual: f64,   // 上記の和と初期値の差 = 数値誤差 (表示)
}
```

- 各ドメイン・結合は自分の変換分を台帳に記帳する(摩擦: kinetic→thermal、
  ジュール: battery→thermal、…)。
- **numerical_residual が数値誤差の見える化**: Baumgarte の偽注入、移流の散逸などが
  ここに現れる。ユーザーが「シミュレーションの正直さ」を常時監視できる。
  検証テストは residual の閾値(シーン規模の 10⁻³ 相対/分)を課す。

### 4.2 エントロピーの観測

- 熱移動: $\Delta S = Q(1/T_{cold} - 1/T_{hot}) > 0$ を熱流ごとに積算。
- 理想気体: $S = nR\ln(V T^{f/2}) + const$ の変化分。
- 表示専用(ダイナミクスには使わない)。「第 2 法則が破れない」ことのリアルタイム証人。

### 4.3 断熱圧縮・膨張(気体⇔力学の結合)

ピストン(Slider ジョイント)が気体区画を圧縮する仕事率 $\dot W = p\,\dot V$。
第 1 法則から $\frac{f}{2}nR\,\dot T = -p\dot V + \dot Q$。
力学側はピストン面に $F = pA$ を受ける。時間積分は世界ステップ内で半陰的
(体積は力学の新位置、温度更新はその後)— エネルギー整合は台帳で検算。

## 5. 適用スケールと限界

- 集中熱容量は $Bi < 0.1$ で誤差 <5%。違反時は自動で警告 + 多ノード化提案。
- 温度範囲: 物性表の有効域(概ね 200–2000 K)。プラズマ・燃焼化学は対象外
  (発熱反応は「熱源スクリプト」として与える。反応速度論は [14-quantum/03](../14-quantum/03-effective-models.md) の
  アレニウス式を熱源に使える)。
- 気体区画は一様(音速平衡の仮定 = 区画サイズ ≪ 音速×タイムステップが崩れる高速現象は不可)。

## 6. 他ドメインとの結合

| 相手 | 内容 |
|---|---|
| 力学 | 摩擦・衝突散逸 → 熱源(受)/ 気体圧力 → ピストン力(送)/ 熱膨張(Phase 5) |
| 流体 | 対流熱伝達(境界)、Boussinesq 浮力(送: 温度場)、水温 |
| 電磁気 | ジュール熱 $I^2R$(受)、熱電・温度依存抵抗(送: 温度) |
| 統計 | 温度の分子論的意味(気体デモとの整合表示)、ブラウン強度 $\propto k_BT$ |

## 7. 検証

- 熱平衡: 2 ノード接触 → 共通温度 $T_{eq} = (C_1T_1 + C_2T_2)/(C_1+C_2)$、機械精度。
- 断熱圧縮: 体積半分 → $T_2 = T_1 (V_1/V_2)^{\gamma-1}$、± 1%(準静的に動かした場合)。
- カルノー限界: 2 熱浴間の任意サイクル機構の効率 $\le 1 - T_c/T_h$(サイクル積分で確認)。
- 台帳: 全シーンで residual < 閾値。エントロピー単調増加(孤立系設定)。

## 8. 実装フェーズ対応

Phase 1: ThermalNode + ニュートン冷却 + 摩擦発熱の記帳 + 台帳。Phase 3: 伝導ネットワーク・多ノード・気体区画。
Phase 4: エンジン/ヒートポンプのデモ機構。Phase 5: 熱膨張・実在気体。

## 10. 性能プロファイル

- ホットスポット: エネルギー台帳の集計(全ドメイン横断)。
- 目標アルゴリズムとオーダー: 台帳集計 $O(N)$。気体区画は $O(1)$。
- SoA レイアウト: ThermalNode(temperature/heat_capacity/heat_accum)を配列化。
- 並列化単位: ノード集計を rayon 分割 + 順序固定リダクション。
- SIMD 対象カーネル: 台帳の内訳集計。
- GPU 適性: 低(ノード数は通常少ない)。
- ベンチ: 多ノードシーンの台帳集計コスト。
