# 検証 03. デモシナリオ集 — 「検証して遊ぶ」の実体

各デモ = シーン JSON + UI 定義 + **合格基準**(目視 + Probe の数値)。
すべて決定論(シード表示・リプレイ可)、エネルギー台帳・近似表示つき。

**TDD での位置づけ**(v2): デモは **Phase B の実装ウェーブ**(旧 Phase 1〜5 = 実装順)ごとの
**ドメイン内スモークテスト**として実装中に使う(単一ドメインで完結する D1 等)。多ドメインデモ
(D18 氷と飲み物など)は **Phase C の結合テスト**で検証する。以下の「Phase 1〜5」見出しは
Phase B 内の実装ウェーブを指す([22-roadmap/01-phases.md](../22-roadmap/01-phases.md))。
天体は **Pα ウェーブ**、レンダリングは **Phase D**。

## UI 共通仕様(全デモ)

1. **再生制御**: 再生/停止/1 step/スロー(×0.01–×1)。時刻・step 表示。
2. **Probe グラフ**: シーン定義の観測量を時系列表示、CSV エクスポート。
3. **保存量パネル**: エネルギー内訳(積み上げ)、residual、P/L(保存判定つき
   [02-conservation-laws.md](02-conservation-laws.md) §4)。
4. **シード・ハッシュ**: シード入力欄、状態ハッシュ表示、リプレイ保存/読込。
5. **近似バッジ**: このシーンの近似一覧(`approximations()` API)。クリックで該当設計書へ。
6. **予測→実験ワークフロー**(できるだけ): 実行前にユーザーが予測値を入力 →
   実測と比較表示(例: D5 の滑り出し角)。「検証して遊ぶ」の体験設計の核。

## 実装上の位置づけ

- デモは `demo/scenes/*.json` + 各シーン固有の最小 UI 拡張。物理は World API のみ使用
  ([20-integration/04](../20-integration/04-world-api.md) §5 の受け入れ条件)。
- 各デモの合格基準は CI のヘッドレステスト(描画なし・Probe 値の assert)として自動化する。
  目視項目(渦の見た目等)のみ手動チェックリスト。
